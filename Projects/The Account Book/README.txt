ТРЕБОВАНИЯ К ПРОГРАММЕ "The Account book"

О ПРОГРАММЕ "The Account book"

Это программа вида "домашняя бухгалтерия".
Она позволяет организовать личные финансы, и представляет собой гроссбух.
Предназначена для домашнего использования.

ТРЕБОВАНИЯ К ИСПОЛНЕНИЮ И ПОСТАВКЕ

Составить код программы на ЯПВУ Java.

ТРЕБОВАНИЯ К ФУНКЦИОНАЛЬНОСТИ

Программа должна предоставлять пользователью GUI, и не должна - CUI (т.к. предназначена для домашнего пользователя).
Программа должна предоставлять свой пользовательский интерфейс сугубо на английском языке
(локализация на языке Исполнителя безусловно снизит популярность Программы).
Программа не должна быть локализована на языке Исполнителя (локализация на языке Исполнителя безусловно снизит популярность Программы).
Программа предоставляет свою функциональность через обычное окно (Заказчику требуется обычное ПО, не веб- приложение).
Окно должно оснащаться следующими элементами управления:
	рамка со строкой заголовка		(The Account book) и кнопкам управления состоянием окна,
	строка меню (с пунктами:		File (New, Open, Save as, Delete, Exit),
						Search (Search, Search Next, Search Previous,),
						Help (About programm));
	панель инструментов:(с инструментами	By summ		- сортирует записи гроссбуха по сумме,
						By start date	- сортирует записи гроссбуха по дате открытия позиции,
						By individual	- сортирует записи гроссбуха по оппозитной стороне позиции);

лист гроссбуха должен быть конечной длины (80 строк); гроссбух должен позволять работать с рядом листов (28 листов).
Программа должна позволять сохранять данные в файлах - как отдельные листы гроссбуха, так и весь гроссбух.
Программа не должна позволять отправлять данные на облачный сервис, или же почтовый ящик (это необходимо для сохранности данных Программы).

ТРЕБОВАНИЯ К ПРОЦЕССУ РАЗРАБОТКИ

Использовать в процессе разработки требования заказчика.
Разбить процесс разработки на ряд этапов:
разработка прототипа продукта				(суть этапа:
							GUI разрабатывать на AWT - в дальнейшем на Swing,
							функциональность меню Программы ограничить выпадением диалоговых окон, и соответствующую логику разрабатывать только на втором этапе процесса;
							панель инструментов Программы не разрабатывать - в интересах дальнейшей разработки на Swing,
							подсистему работы с листами не разрабатывать - в интересах дальнейшей разработки на Swing,
							ограничить число листов гроссбуха одним,
							ограничить число строк на листе 2- мя);

разработка собственно продукта на проэкт. стадии	(перевести GUI Программы на Swing,
							внедрить панель инструментов,
							разработать систему работы с листами - не менее 80 стр. в листе и 28 листов всего,
							доработать подсистему сохранения в файлы - отдельных листов и гроссбуха);

разработка собственно продукта на финаль. стадии	(проверить код программы);

вести разработку на каждой стадии в редакторе Notepad++, программную документацию вести путем комментирования кода программы
(комментирование исполнять по секциям файла исходного кода, использовать однострочные комментарии - 
при необходимости имитировать ими блок многострочного комментария (руководствоваться настоящим указанием на правах принятой для проекта конвенции)).
